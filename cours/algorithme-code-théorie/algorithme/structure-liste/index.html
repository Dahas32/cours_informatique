Structure : liste

Cours/Algorithme, code et théorie/Algorithme/Structure : liste

Prérequis :

  • Complexité en moyenne

Mise en œuvre de la structure de liste, qui est une amélioration du de la structure de tableau.

Tableau

On l'a vu, pour un tableau on a les complexité suivantes :

  • La complexité dans le cas le pire de la création de la structure : O(1)
  • La complexité dans le cas le pire pour trouver l'élément d'indice i : O(1)
  • La complexité dans le cas le pire de l'ajout d'un élément à la structure : O(1) en fin de structure s'il reste de la place, sinon ajout impossible. O(n−i)=O(n) à l'emplacement i.
  • La complexité dans le cas le pire de la suppression d'un élément de la structure : Pour un tableau O(1) en fin de structure. O(n−i)=O(n) à l'emplacement i
  • La complexité dans le cas le pire de la suppression de la structure : O(1)

Cette structure est adaptée lorsque l'on ne doit pas supprimer/ajouter des éléments en milieu de structures. Cependant, on doit connaître a priori le nombre maximum d'éléments.

Liste

Les listes de python se comportent de manières différentes. Tout comme les tableaux ce sont des objets pouvant contenir une succession d'autres objets auxquels on peut accéder par un indice*, mais on peut facilement ajouter/supprimer un nombre infini d'éléments en fin de liste.

Vous devriez savoir manipuler des listes comme personne. Mais si vous avez besoin d'une piqûre de rappel, n'hésitez pas à consulter la partie listes du cours sur les bases du code.

Structure d'un liste

Une liste peut être implémentée de cette façon :

  • on commence par créer un tableau de taille t=k, le nombre initial d'éléments étant n=0.
  • à chaque ajout d'éléments :
    1. test si n<t :
      • si oui :
        • n=n+1.
      • sinon :
        • on alloue un tableau 2×t éléments et t=2×t
        • on copie les n premiers éléments du tableau initial dans le nouveau tableau et on supprime le tableau initial.
        • n=n+1.
    2. décalage de tous les éléments d'indice supérieur au rang de l'ajout et insertion de l'élément.

Complexités

  • La complexité dans le cas le pire de la création de la structure : O(1)
  • La complexité dans le cas le pire pour trouver l'élément d'indice i : O(1)
  • La complexité dans le cas le pire de l'ajout d'un élément à la structure : O(n)
  • La complexité dans le cas le pire de la suppression d'un élément de la structure : O(1)
  • La complexité dans le cas le pire de la suppression de la structure : O(1)

complexité d'ajout de n éléments

Ajouter un élément à la structure peut très mal tomber. Cela peut être juste au moment où l'on doit doubler la taille de la structure. C'est donc de complexité O(n) opération s'il y avait n élément dans la liste au moment de l'ajout.. Mais ensuite, les n−1 suivants ajout vont forcément bien se passer et auront tous une complexité de O(1) opérations.

On a même le résultat suivant :

L'ajout de n éléments à une liste originellement vide prend O(n) opérations au maximum

preuve

Complexité d'ajout de n éléments à une liste :

  • Dans le cas le pire le dernier ajout entraîne un doublement de la taille de la structure : un nouveau tableau est créé en O(1) puis les n éléments de l'ancien tableau sont copiés dans le nouveau en O(n) opérations
  • Le précédent tableau était de taille n−1 et a nécessité O(n) opérations pour être créé puis rempli (recopie de n/2 anciens éléments puis insertion de n/2 nouveaux éléments).
  • Le tableau encore d'avant d'avant était de taille n/2 et son remplissage a pris O(n/2) opérations (recopie de n/4 anciens éléments puis insertion de n/4 nouveaux éléments)
  • Le tableau encore encore d'avant d'avant était de taille n/4 et son remplissage a pris O(n/4) opérations
  • ...
  • le ième tableau précédent était de taille n/2i et son remplissage a pris O(n/2i) opérations
  • ...
  • le log2⁡(n) tableau précédent était de taille n/2log2⁡(n=1 et son remplissage a pris un nombre d'opérations de O(n/2log2⁡(n)=O(1) opérations

La complexité totale est donc de :

C(n)=O(n+n+n2+n4+n8+⋯+1)=O(n+∑i=0log2⁡(n)n2i)=O(n(1+∑i=0log2⁡(n)12i))

Comme ∑i=0+∞12i=1, on a :

(1+∑i=0log2⁡(n)12i))≤(1+∑i=0+∞12i))≤2

Et donc : C(n)=O(n)

Comme la complexité d'ajout d'un élément à une liste n'est pas constante, il nous faut un nouvel outil pour en appréhender sa complexité :

Définition

On appelle complexité amortie d'un algorithme la complexité d'effectuer n fois une opération le tout divisé par n.

Dans le cas d'une structure simple, la complexité amortie est égale à la complexité puisque l'on fait n fois la même chose mais pour des structure plus complexe comme les listes, lorsque l'on ajoute n fois un élément, cette opération n'est coûteuse qu'un petit nombre de fois.

La complexité amortie d'ajout de n éléments dans une liste est alors O(1). En effet :
complexité d’ajout de n éléments dans une liste /n=O(n)/n=O(1)

On considère que La complexité de l'ajout d'un élément en fin de liste est en O(1) opérations.

La complexité amortie est un concept avancé. Il ne faut pas le confondre avec la complexité en moyenne, c'est bien n fois la complexité maximale que l'on considère lorsque l'on effectue les opération successivement. C'est un moyen efficace de calculer la complexité d'un algorithme lorsque l'on utilise des structures dont l'opération coûteuse n'est faite qu'un petit nombre de fois.

Attention

Un piège courant lorsque l'on débute avec les liste en python est d'ajouter un élément en fin de liste avec la commande : l = l + [x]. C'est une erreur car la complexité est beaucoup plus importante que si l'on utilise la méthode append :

  • complexité de l = l + [x] : O(len(l)) car on crée une nouvelle liste !
  • complexité de : l.append(x) : O(1) car on ajoute à la fin d'une liste déjà existante
