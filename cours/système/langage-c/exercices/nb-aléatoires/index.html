Entier aléatoire

  • Toutes les fonctions sont à écrire dans le programme principal, en dehors de la fonctions main.
  • Le programme main doit permettre de tester chaque fonction demandée

Créez une fonction de signature :

int aleatoire_int(int min, int max);

permettant de rendre un entier aléatoire entre min et max inclus (les deux paramètres de la fonction).

Pour cela, vous pourrez utiliser les fonctions (de la libc) suivantes définis dans <stdlib.h> :

  • srand dont le but est d'initialiser l'algorithme de nombres aléatoire avec ue seed. Attention cette fonction n'est à n'utiliser qu'une fois par programme, au tout début.
  • rand qui rend un entier aléatoire entre 0 et et RAND_MAX
  • le modulo (%) qui permet de conserver l'équiprobabilité.

Testez la fonction précédente en faisant la moyenne de 100000 tirage de nombres entre -50 et +50 et en vérifiant pour chaque tirage que l'on est bien entre -50 et 50.

Liste d'entiers aléatoires

Créez une fonction de signature :

int *aleatoire_tab(int max, size_t nombre);

Qui tire nombre nombres aléatoires entre 0 et max (exclu) et les rend dans un tableau de taille nombre. Vous créerez un tableau de nombre entiers int avec une allocation dans le tas.

Testez la fonction précédente avec int *t = aleatoire_tab(10, 1000) et en remplissant le tableau int n[10] tel que n[i] contienne le nombre de fois où le nombre i est présent dans t.

Intervalle aléatoire

Créez une fonction de signature :

double aleatoire_01();

permettant de rendre un réel aléatoire entre 0 et 1.

Probabilité

Utilisez la fonction précédente pour créer une fonction de signature :

int aleatoire_01(double proba);

Qui rend 1 avec une probabilité proba et 0 sinon.

Utilisez la fonction précédente pour créer une fonction de signature :

int *aleatoire_01_liste(double proba, size_t nombre);

Qui rend une liste de taille nombre contenant des 0 ou des 1 tirés selon une probabilité proba.

Testez la fonction précédente en tirant une liste de taille 100 avec une probabilité de .1, .5. et .75. Vérifier que les proportions sont bien vérifiées.

Mélange d'un tableau

Créez une fonction qui mélange les $n$ premiers éléments d'un tableau d'entier. Sa signature doit être :

void nb_chiffres(int *t, size_t n);

Vous utiliserez l'algorithme de mélange de Knuth, improprement appelé algorithme de Fisher-Yates par les mathématiciens.

Testez la fonction précédente en tirant mélangeant plusieurs fois le tableau contenant les 10 premiers entiers et en affichant le résultat.

Compilation séparée

Le but de cet exercice est de comprendre la compilation séparée, tout en jouant avec les nombres.

Décomposez le code en deux unités fonctionnelles :

  • le programme principal (fichier main.c)
  • le module aléatoire (deux fichiers aleatoire.c et aleatoire.h)

Créez un fichier Makefile pour gérer la compilation de ce projet.
