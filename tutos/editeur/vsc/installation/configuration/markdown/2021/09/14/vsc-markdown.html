extensions génériques

La documentation comporte une partie consacrée au markdown. Nous allons utiliser deux extensions :

  • Markdown All in One qui permet de fluidifier l’écriture de markdown et permet un export de celui-ci en html.
  • markdownlint qui souligne en orange les fautes de style (un espace après un . par exemple) et les erreurs de markdown (sauf dans la hiérarchie des paragraphes par exemple).

Le seconde extension est un linter, qui incite à utiliser les bonnes pratiques d’écriture. Il en existe pour quasi tous les langages.

export et preview

Pour exporter le markdown dans quelque chose de plus joli. On peut utiliser la palette de commande pour exétuter les commandes :

  • markdown All in One: Print current document to HTML : qui va rednre un fichier html
  • markdown: Open preview pour faire une preview.

extensions specifiques

  • Markdown Preview Enhanced Permet d’avoir des feuilles de styles jolies lorsque l’on converti du markdown.
